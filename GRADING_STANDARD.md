# Python 程序设计大作业公开评分标准

总分 100 分。评分重点是：是否完成一个结构清晰、可运行、可测试、同时支持 CLI 和 Streamlit 的 Python 数据应用。

## 分数上限

| 等级 | 说明 | 分数上限 |
|---|---|---:|
| A 基础版 | CLI 能完成基本增删查改，JSON 能保存 | 75 |
| B 标准版 | CLI + Streamlit 都可用，且共用业务逻辑 | 88 |
| C 完整版 | 有搜索、过滤、统计、pytest、异常处理 | 95 |
| D 挑战版 | 有图表/导入导出/示例数据/完整报告/AI review 记录 | 100 |

## 评分细则

| 模块 | 分值 | 说明 |
|---|---:|---|
| 功能完整性 | 25 | 主题明确，至少包含新增、查看、搜索/过滤、更新/删除、统计 |
| CLI 入口 | 12 | 使用 argparse 或等价标准方式，子命令清晰，错误提示友好 |
| Streamlit 页面 | 15 | 页面可运行，能完成核心操作，不只是展示静态文字 |
| 数据建模与持久化 | 15 | 使用 dataclass 或清晰模型，JSON 读写可靠，数据结构合理 |
| 模块化结构 | 8 | CLI、页面、业务逻辑、存储、分析分层清楚 |
| 异常处理与健壮性 | 8 | 能处理空数据、坏输入、文件不存在、JSON 损坏等常见问题 |
| pytest 测试 | 10 | 至少覆盖模型、存储、业务逻辑和统计分析的关键路径 |
| 文档与报告 | 5 | README、使用示例、AI 反思、最终报告材料完整 |
| Git 与提交规范 | 2 | 提交历史清楚，最终提交信息包含 `提交作业` |

## 不建议或不接受

- 只做 Streamlit 页面，没有 CLI：不能超过 78 分。
- 只做 CLI，没有 Streamlit：不能超过 75 分。
- 使用 Tkinter 作为最终交付：不作为有效展示层评分。
- 使用 Notebook 作为最终交付：不作为有效展示层评分。
- 主要业务逻辑全部堆在 `app.py`：模块化项会明显扣分。
- 无法运行或无法安装：原则上不能超过 60 分。
- 无测试：原则上不能超过 85 分。
